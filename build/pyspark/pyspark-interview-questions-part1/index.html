1. What is PySpark and how is it different from Pandas?‚Äã

PySpark is the Python API for Apache Spark, an open-source distributed computing framework designed to process large-scale data across multiple nodes in a cluster. It provides an interface for leveraging Spark‚Äôs capabilities (such as distributed data processing, fault tolerance, and in-memory computation) using Python syntax.

‚úÖKey Differences Between PySpark and Pandas‚Äã

FeaturePySparkPandas
Data SizeHandles massive datasets (terabytes or petabytes) distributed across multiple machinesWorks with small to medium datasets that fit in a single machine‚Äôs memory
Execution ModelLazy evaluation ‚Äî transformations are only executed when an action (like .count() or .show()) is calledEager evaluation ‚Äî operations execute immediately
PerformanceOptimized for parallel, distributed processing via Spark‚Äôs execution engineSingle-threaded and runs on a single machine
EnvironmentRequires a Spark cluster or local Spark setupRuns locally with no cluster requirement
Use CasesBig Data ETL, Data Engineering, Machine Learning Pipelines on distributed dataData analysis, exploration, and quick prototyping

Example:

# Pandas example
import pandas as pd
df = pd.read_csv("sales.csv")
df.groupby("region")["revenue"].sum()

# PySpark example
from pyspark.sql import SparkSession
spark = SparkSession.builder.appName("SalesApp").getOrCreate()
df = spark.read.csv("sales.csv", header=True, inferSchema=True)
df.groupBy("region").sum("revenue").show()

üí¨ Summary:

  • Pandas is like a powerful notebook ‚Äî excellent for data that fits in memory.
  • PySpark is like a distributed supercomputer ‚Äî designed to handle big data efficiently and fault-tolerantly.

2. What is SparkSession in PySpark?‚Äã

SparkSession is the entry point to programming with PySpark. It provides a unified interface to interact with all Spark functionalities, including DataFrame and SQL APIs, streaming, and machine learning.

2.1 How it replaced older contexts?‚Äã

  • Introduced in Spark 2.0, replacing older contexts (SparkContext, SQLContext, and HiveContext).
  • Acts as the main gateway to access Spark‚Äôs core features.
  • Handles the configuration, resource management, and session state for the Spark application.

2.2 What is the Core Responsibilities of SparkSession?‚Äã

  1. Create and manage DataFrames

    df = spark.read.csv("data.csv", header=True, inferSchema=True)

  2. Run SQL queries

    spark.sql("SELECT * FROM customers WHERE age > 30").show()

  3. Access the Spark Context

    sc = spark.sparkContext

  4. Manage configurations and catalogs

    spark.conf.set("spark.sql.shuffle.partitions", 50)
    spark.catalog.listTables()

2.3 Explain the internal architecture of SparkSession in PySpark. How does it interact with SparkContext and SQLContext?‚Äã

  • SparkSession internally creates or uses a SparkContext to connect with the cluster.

  • It also manages:

    • The SQLContext (for structured queries)
    • The Catalog (for metadata and table management)

üßæ Example:

from pyspark.sql import SparkSession

# Create SparkSession
spark = SparkSession.builder \
    .appName("ExampleSession") \
    .config("spark.some.config.option", "some-value") \
    .getOrCreate()

# Load data
df = spark.read.json("data.json")

# Perform SQL operations
df.createOrReplaceTempView("people")
spark.sql("SELECT name FROM people WHERE age > 30").show()

üí¨ Summary:

  • SparkSession = Unified Entry Point for all Spark operations.
  • Simplifies working with structured data, SQL, and configuration.
  • Without it, no PySpark job can run.

3. What is DataFrame and How do you create a DataFrame in PySpark?‚Äã

In PySpark, a DataFrame is a distributed collection of data organized into named columns, similar to a relational table in SQL or a DataFrame in Pandas ‚Äî but capable of handling data at scale across a cluster.

‚úÖYou can create a DataFrame in PySpark in three main ways:‚Äã

1Ô∏è‚É£ From a Structured Data Source (like CSV, JSON, Parquet, etc.)‚Äã

Using the spark.read API:

from pyspark.sql import SparkSession

spark = SparkSession.builder.appName("DataFrameExample").getOrCreate()

# Read from CSV
df_csv = spark.read.csv("orders.csv", header=True, inferSchema=True)

# Read from JSON
df_json = spark.read.json("orders.json")

df_csv.show(5)

Explanation:

  • header=True ‚Üí tells Spark to use the first row as column names
  • inferSchema=True ‚Üí automatically detects data types
  • These files are distributed and loaded in parallel across worker nodes

2Ô∏è‚É£ From a Python Collection (List, Dictionary, or RDD)‚Äã

You can manually create a small DataFrame using local Python data ‚Äî useful for testing or demos:

data = [("Alice", 25), ("Bob", 30), ("Catherine", 29)]
columns = ["Name", "Age"]

df_manual = spark.createDataFrame(data, columns)
df_manual.show()

Or from an RDD:

rdd = spark.sparkContext.parallelize(data)
df_rdd = rdd.toDF(columns)

3Ô∏è‚É£ From an External Database or Data Warehouse‚Äã

You can connect to JDBC sources such as MySQL, PostgreSQL, or Snowflake:

df_db = spark.read.format("jdbc") \
    .option("url", "jdbc:mysql://localhost:3306/sales_db") \
    .option("dbtable", "orders") \
    .option("user", "root") \
    .option("password", "mypassword") \
    .load()

4. Explain the difference between RDD and DataFrame?‚Äã

In PySpark, both RDDs and DataFrames are fundamental abstractions for working with data ‚Äî but they differ significantly in abstraction level, performance, and usability.

FeatureRDD (Resilient Distributed Dataset)DataFrame
DefinitionA low-level distributed collection of objects; the fundamental Spark data structure.A distributed collection of rows with named columns ‚Äî built on top of RDDs.
Abstraction LevelLow-level API ‚Äî requires manual transformations and actions.High-level API ‚Äî designed for structured data operations.
Data Structure TypeUnstructured ‚Äî can hold any type of Python, Scala, or Java objects.Structured ‚Äî data is organized in tabular form like a SQL table.
SchemaNo schema; user must manage data types manually.Has an explicit schema (column names and data types).
Ease of UseComplex to code; requires more lines and transformations.User-friendly ‚Äî supports SQL-like operations and DataFrame APIs.
OptimizationNo automatic optimization; user must handle logic.Automatically optimized by Spark‚Äôs Catalyst Optimizer.
PerformanceSlower due to lack of optimization and serialization overhead.Faster due to Tungsten execution engine and optimized query planning.
Data RepresentationDistributed collection of Java/Python objects.Distributed collection of Rows with named columns.
Use CaseBest for low-level transformations, custom computations, or when schema is unknown.Best for structured data processing, ETL, and SQL operations.
InteroperabilityHarder to integrate with SQL or MLlib directly.Integrates seamlessly with Spark SQL, MLlib, and GraphFrames.
API TypeFunctional (map, flatMap, filter, reduce).Declarative (select, filter, groupBy, join).
Example Syntaxrdd.filter(lambda x: x > 5)df.filter(df.value > 5) or spark.sql("SELECT * FROM table WHERE value > 5")

üí¨Summary:

  • ‚ÄúRDDs give you full control but demand more effort.
  • DataFrames give you structured power and performance optimization with minimal code.‚Äù

5. How do you check the version of PySpark?‚Äã

You can check the PySpark version using any of the following methods, depending on your environment.

1Ô∏è‚É£ Check version using PySpark shell or script‚Äã

import pyspark
print(pyspark.__version__)

Output Example:

3.5.2

This prints the version of the installed PySpark package in your environment.

2Ô∏è‚É£ Check version from SparkSession‚Äã

If you‚Äôve already created a SparkSession:

from pyspark.sql import SparkSession

spark = SparkSession.builder.appName("VersionCheck").getOrCreate()
print(spark.version)

Output Example:

3.5.2

Here, spark.version gives the version of the Spark runtime (the engine your code is actually running on).

Sometimes, the installed PySpark version (pyspark.__version__) and Spark runtime version (spark.version) may differ slightly if your environment is misconfigured ‚Äî checking both is a good practice.

3Ô∏è‚É£ Check via Command Line (CLI)‚Äã

If you‚Äôre using a local or cluster terminal:

pyspark --version

Output Example:

      /_/
Using Scala version 2.12.15, Java HotSpot(TM) 64-Bit Server VM, 11.0.22

This shows both the Spark version and Scala version used by your PySpark installation.
